Purpose and Relevance to Petition

The petition asserts that Mr. Okedi is well-positioned to advance his proposed endeavor — specifically, that he has a concrete forward-looking research plan and demonstrated commitment to building BioForge, an integrated computational-experimental platform for optimizing domestic waste-to-value bioprocesses. Under Matter of Dhanasar, evidence that a petitioner is well-positioned includes not only past accomplishments but also demonstrated commitment through concrete steps already taken.

This exhibit provides that evidence. It documents that BioForge is not a speculative future project, but a real software system currently in active development, with working code, a validated output tied directly to the petitioner's published research, and a public repository available to any U.S. institution that wishes to use or build upon it.

Core Statement for the Record
BioForge is currently in active development. The core optimization engine has been prototyped, implemented in Python, and validated against experimental data from the petitioner's flagship peer-reviewed publication. The repository was initialized in March 2026, contains 15 source files and over 1,100 lines of working code, and is publicly accessible at https://github.com/ogaga-ai/BioForge. Every claim made about BioForge in the petition narrative corresponds to a specific implemented file or working output documented in this exhibit.

How to Independently Verify BioForge

Any reviewer may independently verify the existence and functionality of BioForge using the following steps:

  • View the repository: Navigate to https://github.com/ogaga-ai/BioForge in any web browser. The commit history, file structure, source code, and README are publicly accessible without an account.
  • Verify authorship: All commits are attributed to Ogaga Maxwell Okedi. The commit timestamp confirms repository creation in March 2026.
  • Run the demonstration: Clone the repository, install dependencies (pip install -r requirements.txt), and run the Jupyter notebook (notebooks/BioForge_CitricAcid_Optimization.ipynb). The notebook will reproduce the citric acid optimization workflow and generate protocol card output.
  • Inspect the source code: All Python modules in bioforge/ are documented, executable, and written in standard Python 3.11 using widely available open-source libraries.
Note on Development Stage
BioForge is deliberately documented as a prototype (v0.1) rather than a finished product. This accurately reflects the early-career stage of Mr. Okedi's proposed endeavor. The petition does not claim that BioForge is complete — it claims that Mr. Okedi has begun building it and is well-positioned to complete it. This exhibit provides exactly that evidence: a real, working foundation that proves the work has started, the methodology is sound, and the deliverables are achievable.
